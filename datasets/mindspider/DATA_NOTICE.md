# Data notice — MindSpider datasets

## Demo data (--source demo)
All data in this directory when generated via `--source demo` is
synthetically generated by Cortex and does not contain real social
media posts or personally identifiable information (PII).

## Live data (--source mysql)
When connecting to a live MindSpider deployment via --source mysql:
- Data is processed locally and NEVER transmitted to external services
- Cortex does not store PII from social media platforms
- Topic-level aggregations only — individual post attribution is not retained
  beyond the sample_posts field (limited to 3 excerpts per topic)
- Data retention: today_topics.json is overwritten on each run. No historical
  accumulation unless the user explicitly archives files.
- Compliance: users are responsible for ensuring their MindSpider deployment
  complies with platform terms of service and applicable data protection laws
  (GDPR, DPDP Act 2023, PIPL, etc.)

## Data handling in evaluation pipelines
When MindSpider data is used as evidence in research_pipeline.py:
- Data is passed to the Anthropic API for analysis (subject to Anthropic's
  data handling policy: https://www.anthropic.com/policies)
- No MindSpider data is stored in the knowledge/store.py FTS5 database
  unless the user explicitly runs the indexer
- Evolution log entries (evolution_log.jsonl) never contain raw social data —
  only skill fitness scores and mutation metadata
